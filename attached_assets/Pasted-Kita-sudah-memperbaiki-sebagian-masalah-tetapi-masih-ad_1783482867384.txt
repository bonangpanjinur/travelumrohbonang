Error kedua:

Supabase REST API mengembalikan

400 Bad Request

untuk query:

select=*,
package:packages(title),
departure:package_departures(departure_date)

Saya ingin kamu melakukan audit penuh.

LANGKAH 1

Cari semua query Supabase yang menggunakan:

.select()

atau

select=

Periksa apakah ada:

package:packages(...)
departure:package_departures(...)

Pastikan relationship tersebut benar-benar ada di database.

Jika foreign key tidak ada,
ubah query supaya tidak memakai embedded relationship.

Misalnya ubah sementara menjadi:

.select("*")

agar aplikasi bisa berjalan.

LANGKAH 2

Cari endpoint berikut:

/api/auth/user

Tambahkan logging lengkap.

Misalnya:

console.error("AUTH USER ERROR");
console.error(error);
console.error(error.stack);

Saya ingin mengetahui exception sebenarnya.

LANGKAH 3

Cari semua endpoint yang menghasilkan HTTP 500.

Tambahkan logging:

console.error(routeName, error);

supaya stack trace muncul di Runtime Logs Vercel.

LANGKAH 4

Audit schema database.

Pastikan tabel berikut benar-benar ada:

- bookings
- packages
- package_departures
- notifications
- faqs

Pastikan nama tabel sama persis.

Pastikan foreign key:

bookings.package_id -> packages.id

bookings.departure_id -> package_departures.id

Jika tidak ada, jelaskan.

LANGKAH 5

Cari semua penggunaan:

.single()

atau

.maybeSingle()

yang dapat melempar exception.

Gunakan pengecekan error terlebih dahulu.
