Lakukan audit menyeluruh terhadap backend project ini.

Kondisi saat ini:

Frontend berhasil dibuild dan berjalan di Vercel.

Namun seluruh endpoint backend mengembalikan HTTP 500, antara lain:

- /api/auth/user
- /api/faqs
- /api/packages
- /api/packages/filter-options

Frontend juga menampilkan:

- VITE_TURNSTILE_SITE_KEY is not defined (warning)
- VITE_SENTRY_DSN is not defined (warning)

Tetapi saya yakin kedua warning tersebut bukan penyebab utama.

Saya ingin kamu menemukan akar penyebab HTTP 500.

Lakukan langkah berikut secara berurutan:

1. Audit file yang membuat Supabase client.
   - Cari semua createClient().
   - Pastikan backend menggunakan:
       process.env.SUPABASE_URL
       process.env.SUPABASE_SERVICE_ROLE_KEY
     BUKAN import.meta.env.

2. Cari semua penggunaan:
   process.env
   import.meta.env
   VITE_
   SUPABASE_

3. Pastikan backend TIDAK menggunakan:
   VITE_SUPABASE_ANON_KEY
   VITE_SUPABASE_URL

4. Tambahkan logging startup:

console.log({
  SUPABASE_URL: !!process.env.SUPABASE_URL,
  SERVICE_ROLE: !!process.env.SUPABASE_SERVICE_ROLE_KEY,
  NODE_ENV: process.env.NODE_ENV
})

Jangan pernah mencetak isi key, cukup true/false.

5. Audit setiap endpoint:

- /api/auth/user
- /api/faqs
- /api/packages
- /api/packages/filter-options

Tambahkan try/catch sehingga jika terjadi exception akan muncul:

console.error(error)

beserta stack trace.

6. Cari apakah ada query Supabase yang gagal.

Misalnya:

const { data, error }

Jika error ada,
console.error(error)

7. Cari apakah ada penggunaan

throw error

yang menyebabkan Express langsung mengembalikan 500.

8. Pastikan response error berisi pesan yang jelas.

Misalnya:

res.status(500).json({
  success:false,
  message:error.message,
  stack:process.env.NODE_ENV==="development"
      ? error.stack
      : undefined
})

9. Periksa apakah ada file server yang hanya berjalan di Replit tetapi gagal di Vercel.

10. Jangan melakukan refactor besar.

Fokus hanya menemukan penyebab HTTP 500.

Setelah selesai, berikan laporan:

- File yang menyebabkan masalah
- Baris kode penyebab
- Kenapa bisa menghasilkan 500
- Perbaikan yang dilakukan
