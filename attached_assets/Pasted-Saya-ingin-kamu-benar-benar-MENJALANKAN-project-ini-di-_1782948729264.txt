Saya ingin kamu benar-benar MENJALANKAN project ini di environment Replit, bukan hanya menganalisis source code.

Tujuanmu adalah membuat seluruh aplikasi berjalan dengan sukses di Replit sehingga seluruh masalah pada codebase, database, konfigurasi, dan environment dapat terdeteksi secara nyata.

Lakukan langkah-langkah berikut secara berurutan.

## 1. Audit Environment

* Analisis struktur workspace.
* Identifikasi frontend, backend, API, database, shared package, migration, dan konfigurasi.
* Identifikasi framework dan tools yang digunakan.
* Jangan mengubah arsitektur tanpa alasan.

---

## 2. Install Semua Dependency

Install seluruh dependency menggunakan pnpm sesuai workspace.

Jika ada package yang hilang, install.

Jika ada dependency conflict, selesaikan.

Jangan berhenti sampai seluruh dependency berhasil diinstall.

---

## 3. Setup Environment

Periksa seluruh file konfigurasi seperti:

* .env
* .env.example
* package.json
* pnpm-workspace.yaml
* tsconfig
* vite.config
* drizzle.config
* prisma
* docker
* konfigurasi database

Jika ada konfigurasi yang kurang, buatkan konfigurasi minimal agar project dapat dijalankan di Replit.

---

## 4. Jalankan Database

Deteksi database yang digunakan.

Jika PostgreSQL:

* gunakan PostgreSQL bawaan Replit bila tersedia
* atau buat database baru
* hubungkan project ke database tersebut

Kemudian:

* jalankan migration
* jalankan seed bila tersedia
* pastikan seluruh tabel berhasil dibuat

Jika migration gagal:

* cari penyebabnya
* perbaiki SQL atau migration
* jalankan kembali

---

## 5. Jalankan Backend

Start backend.

Perbaiki semua error runtime yang muncul.

Jangan hanya memperbaiki compile error.

Pastikan server benar-benar berjalan.

---

## 6. Jalankan Frontend

Start frontend.

Pastikan frontend dapat mengakses backend.

Pastikan API tidak gagal.

Pastikan seluruh halaman dapat dimuat.

---

## 7. Audit Database dan Codebase

Bandingkan seluruh schema database dengan codebase.

Temukan:

* tabel hilang
* kolom hilang
* migration rusak
* SQL tidak sesuai
* foreign key salah
* enum salah
* query yang gagal

Perbaiki semuanya hingga backend dapat berjalan tanpa database error.

---

## 8. Perbaiki Semua Error

Perbaiki:

* runtime error
* TypeScript error
* SQL error
* migration error
* import error
* build error
* dependency error
* environment error

Jangan menggunakan solusi sementara seperti any, ts-ignore, atau menonaktifkan strict mode.

---

## 9. Verifikasi

Jalankan kembali hingga seluruh proses berhasil:

* install
* database
* migration
* backend
* frontend
* typecheck
* build

Jika masih ada error, lanjutkan memperbaiki sampai selesai.

---

## 10. Laporan

Berikan laporan lengkap yang berisi:

* dependency yang diinstall
* database yang digunakan
* migration yang dijalankan
* SQL yang diperbaiki
* file yang diubah
* error yang ditemukan
* error yang diperbaiki
* status backend
* status frontend
* status database
* status build
* status typecheck

Jangan berhenti setelah menemukan satu error. Terus jalankan, perbaiki, dan ulangi sampai aplikasi benar-benar berjalan di environment Replit dan siap di-deploy ke Vercel.
