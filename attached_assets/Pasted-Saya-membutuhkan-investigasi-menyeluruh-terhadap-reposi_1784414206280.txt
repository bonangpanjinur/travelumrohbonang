Saya membutuhkan investigasi menyeluruh terhadap repository ini.

## KONDISI

Repository ini sebelumnya dapat di-deploy ke Vercel dan berjalan normal.

Setelah beberapa perubahan dilakukan di Replit, setiap deploy ke Vercel mengalami masalah.

Frontend berhasil di-load, tetapi seluruh endpoint backend gagal.

Console Browser menunjukkan:

- GET /api/admin/menu-permissions/my -> 500
- GET /api/admin/bookings -> 500
- GET /api/admin/packages -> 500
- GET /api/admin/masterdata/* -> 500
- GET /api/cms/site-settings -> 500

Selain itu terdapat:

Supabase REST -> 401 Unauthorized

Saya TIDAK ingin kamu berasumsi.
Saya ingin investigasi berdasarkan source code repository.

====================================================

TUGAS

Cari akar penyebab sebenarnya.

Lakukan audit terhadap:

1. Seluruh proses pembacaan Environment Variables.

2. Seluruh inisialisasi Supabase.

3. Seluruh middleware Auth.

4. Seluruh helper ENV.

5. Startup Express.

6. Semua endpoint /api/admin/*.

====================================================

SECARA KHUSUS PERIKSA

- Apakah process.env dibaca dengan benar.

- Apakah ada ENV yang berubah nama.

- Apakah ada fallback ke ENV Replit.

- Apakah ada penggunaan:
  REPL_ID
  REPLIT_DB_URL
  REPLIT_DEV_DOMAIN
  REPL_OWNER
  atau ENV Replit lainnya.

- Apakah createClient() menggunakan ENV yang benar.

- Apakah SERVICE_ROLE_KEY pernah menjadi undefined.

- Apakah DATABASE_URL pernah menjadi undefined.

- Apakah SUPABASE_URL pernah menjadi undefined.

- Apakah backend menggunakan anon key ketika seharusnya menggunakan service role key.

====================================================

JANGAN HANYA MEMBACA SOURCE.

Tambahkan logging sementara pada startup server untuk mencetak (tanpa membocorkan secret):

✓ apakah ENV ditemukan
✓ panjang string key (misal 215 karakter)
✓ apakah URL valid
✓ apakah createClient berhasil
✓ apakah koneksi ke Supabase berhasil
