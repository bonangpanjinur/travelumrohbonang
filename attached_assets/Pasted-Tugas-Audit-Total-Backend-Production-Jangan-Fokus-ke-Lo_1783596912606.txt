Tugas: Audit Total Backend Production (Jangan Fokus ke Local)

Konteks:

Project ini berjalan dengan arsitektur:

- Frontend : Vercel
- Backend : Express.js
- Database : Supabase PostgreSQL
- Auth : Supabase
- Development : Replit

Masalah utama:

Di Replit semua endpoint berjalan normal.

Namun setelah deploy ke production, banyak endpoint mengembalikan:

- HTTP 500
- HTTP 400
- Failed fetch
- Unauthorized
- Internal Server Error

Saya tidak ingin Anda hanya memastikan aplikasi berjalan di Replit.

Saya ingin Anda menganggap Replit hanyalah environment development.

Target sebenarnya adalah production.

Lakukan audit menyeluruh.

---

Yang HARUS diperiksa

1. Environment Variables

Periksa semua penggunaan:

- SUPABASE_URL
- SUPABASE_ANON_KEY
- SUPABASE_SERVICE_ROLE_KEY
- DATABASE_URL
- JWT_SECRET
- FRONTEND_URL
- CORS_ORIGIN
- NODE_ENV

Pastikan:

- tidak ada typo
- tidak ada env yang undefined
- tidak ada fallback yang salah
- startup server akan gagal dengan pesan jelas jika env wajib tidak ada

Jangan pernah menggunakan nilai kosong.

---

2. Jangan Asumsikan Service Role Salah

Saya sudah berkali-kali mendapat diagnosa bahwa Service Role salah.

Saya ingin bukti.

Tambahkan logging yang menunjukkan:

- apakah env berhasil dibaca
- panjang key
- prefix key
- apakah Supabase client berhasil dibuat
- hasil test sederhana ke database

Jika gagal, tampilkan penyebab sebenarnya.

---

3. Audit Semua Endpoint

Untuk setiap endpoint:

- apakah request masuk
- apakah body diterima
- apakah auth berhasil
- apakah Supabase berhasil dipanggil
- query apa yang dijalankan
- hasil query
- response yang dikirim

Saya ingin tahu tepat di langkah mana request gagal.

---

4. Tambahkan Error Logging Lengkap

Jangan lagi mengembalikan:

Internal Server Error

Saya ingin log berisi:

- stack trace
- endpoint
- method
- body
- params
- query
- Supabase error
- Postgres error
- error code
- detail

---

5. Audit CORS

Pastikan:

- Origin Vercel diizinkan
- OPTIONS request benar
- credentials sesuai
- Authorization header tidak diblok

---

6. Audit Build Production

Pastikan:

- tidak ada import yang hanya bekerja di Replit
- path file tidak bergantung pada filesystem Replit
- process.cwd() aman
- dotenv tidak dipakai secara keliru di production
- semua env dibaca dari process.env

---

7. Audit Supabase

Periksa:

- apakah URL benar
- apakah Service Role benar-benar digunakan di backend
- apakah RLS menyebabkan query gagal
- apakah tabel benar-benar ada
- apakah schema benar
- apakah permission sesuai

Jangan menebak.

Buktikan dengan query test.

---

8. Startup Diagnostic

Saat server hidup tampilkan:

✓ Environment loaded

✓ Supabase connected

✓ Database reachable

✓ Routes registered

✓ Auth initialized

✓ CORS configured

Jika ada yang gagal, hentikan startup.

---

9. Production Health Check

Buat endpoint:

GET /health

yang mengembalikan:

- uptime
- node version
- env
- database status
- supabase status
- jumlah env yang berhasil dimuat
- timestamp

---

10. Root Cause Analysis

Saya tidak ingin patch sementara.

Saya ingin menemukan akar masalah.

Untuk setiap error:

- jelaskan penyebab
- jelaskan mengapa hanya terjadi di production
- jelaskan mengapa tidak muncul di Replit
- berikan solusi permanen

---

Aturan Penting

Jangan mengatakan:

"Service Role kemungkinan salah"

atau

"Supabase belum terhubung"

tanpa bukti log.

Semua kesimpulan harus didukung hasil audit.

Fokus utama adalah deployment production, bukan environment Replit.

Tujuan akhir adalah backend berjalan stabil di Vercel + Supabase dengan logging yang memudahkan diagnosis jika terjadi kegagalan lagi.
