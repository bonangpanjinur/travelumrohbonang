Audit tahap pertama sudah selesai.

Sekarang saya ingin menemukan penyebab pasti HTTP 500 di Vercel.

JANGAN melakukan refactor.
JANGAN mengubah business logic.
JANGAN mengubah database.

Fokus hanya mencari akar masalah.

1. Audit Semua Error 500

Cari seluruh kode yang dapat menghasilkan HTTP 500.

Contoh:

throw error
next(error)
catch(error)
res.status(500)

Buat laporan:

File

Line

Route

Kondisi yang menyebabkan 500
2. Audit Semua Client Supabase

Cari semua

createClient(...)

Pastikan hanya ada SATU implementasi.

Jika ada lebih dari satu:

laporkan semuanya.

Periksa juga apakah ada yang memakai

anon key
service role key
user session
3. Audit Semua Middleware Auth

Untuk setiap middleware tampilkan flow.

Misalnya

Request

↓

Read Cookie

↓

Read Authorization Header

↓

Verify JWT

↓

Get User

↓

Next()

Jika salah satu bisa menghasilkan 500,

laporkan.

4. Audit Semua Route Admin

Cari semua route:

/api/admin/*

Untuk setiap route tampilkan:

Route

↓

Middleware

↓

Service

↓

Supabase Query

↓

Response

Saya ingin mengetahui titik mana yang gagal.

5. Audit Semua Query Supabase

Jika ada kode seperti:

if(error){
    throw error;
}

laporkan.

Jangan diperbaiki.

Saya ingin tahu file dan line number.

6. Tambahkan Diagnostic ID

Setiap request otomatis memiliki ID.

Misalnya

↓

GET /api/admin/categories

↓

middleware auth

↓

permission

↓

supabase

↓

500

Sehingga saya bisa mencocokkan log dengan request.

7. Tangkap Error Asli

Saat terjadi HTTP 500,

console harus mencetak lengkap:

Method

Error Name

Error Message

Stack

Supabase Error Code

Supabase Message

SQL State (jika ada)

Internal Server Error
8. Audit Environment Variable yang Dipakai

Saya tidak ingin hanya tahu env tersedia.

Saya ingin tahu env mana yang benar-benar digunakan.

Contoh:

SUPABASE_URL

Digunakan oleh:
src/lib/supabase.ts

Status:
Loaded

Lakukan untuk semua env.

9. Audit Vercel Runtime Compatibility

Periksa apakah ada penggunaan:

filesystem write
process.cwd()
child_process
worker thread
websocket server
long-running server

yang tidak cocok dengan Vercel Serverless.

Laporkan jika ada.
